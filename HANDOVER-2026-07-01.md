# HANDOVER — oracle-var-resolver (2026-07-01)

Oracle 프로시저의 변수/동적쿼리를 브라우저에서 실제 값으로 바꿔주는 정적 웹앱.
배포: GitHub Pages — https://hannimman.github.io/oracle-var-resolver/ (repo: hannimman/oracle-var-resolver)
현재 버전: **v1.5.3**

---

## 작업 요약

이번 세션에서 0에서 시작해 아래를 만들고 배포까지 완료:

1. **치환 모드(기본)** — 쿼리 + 변수 초기화 블록을 붙여넣으면 Oracle 날짜식을 브라우저에서 계산해 변수를 SQL 리터럴로 치환.
   - 지원 함수: `TO_DATE`(포맷 생략 가능/추론), `TO_CHAR`, `LAST_DAY`, `ADD_MONTHS`, `SUBSTR`, `REPLACE`, `TO_NUMBER`, `NVL`, `COALESCE`, `TO_TIMESTAMP`
   - 연산: 날짜 ± 정수(일), `||` 문자열연결, 괄호식, 단항 부호
   - 마스크 대소문자 무시, 구분자 없는 문자열(`'20260601'`)도 매칭
   - 대입형태: `VAR := EXPR;`, `SELECT ... INTO VAR FROM DUAL`(다중 INTO 포함), 테이블 조회 INTO는 "수동입력"칸
   - `--`, `/* */` 주석 제거
   - 날짜 리터럴: 시:분:초 있으면 `TO_DATE(... HH24:MI:SS)`, 자정이면 `TO_DATE('YYYY-MM-DD')`
   - 입력변수 따옴표 자동 처리(넣든 안 넣든 `'202606'`)
   - 변수 대소문자 무시 매칭(선언 `p_x` ↔ 쿼리 `P_X`)
   - 초기화 블록 없이 쿼리만 → `P_`/`V_`(+ 사용자 등록 프리픽스/이름) 자동 추출
   - `①쿼리` 옆 ＋팝업: 커스텀 프리픽스/변수명 등록·삭제(localStorage)
   - 추천 버튼: 월/날짜(이번달/전달/전전달), 코드값(BIZ_SECT=유아/초등/중학/고등, DEPT_CD=상담/관리, CRTR_ID=test)

2. **동적 SQL 조립 모드(자동 감지)** — `V_SQL := V_SQL || '...'` + `IF/ELSIF/ELSE/END IF` 블록.
   - `①쿼리`에 붙여넣고 분석하면 `:=` + (`||` 또는 `IF..THEN`) 감지해 자동 전환
   - 입력값에 맞는 분기만 조립, IF 조건값을 기본값으로 프리필, `''||var||''` 삽입, append마다 줄바꿈 유지
   - FOR/LOOP는 미지원(건너뜀)

3. **공통** — "빈 줄 제거" 토글 스위치(기본 ON, DBeaver 빈줄-구분자 오류 방지), 결과 복사(클립보드 + `file://` execCommand 폴백 + 결과창이 버튼으로 빨려드는 애니메이션), 초기화, 좌상단 수동 버전표기.

4. **배경 테마** — 우하단 토글로 오로라 ↔ 마그마 라바 무드등 전환(localStorage 저장). 마우스/클릭 반응.

## 성공/실패 기록

### 통한 접근
- **평가기 재사용**: 치환 모드의 `evalExpr`(app.js)를 동적 모드가 그대로 재사용 → `||`/변수삽입 계산 중복 제거.
- **Oracle TO_DATE 관대함**: 마스크의 구분자를 `[^0-9]?`로 처리해 `'20260601'`을 `'YYYY-MM-DD'`로도 매칭(실제 Oracle 동작과 일치).
- **암묵적 형변환**: 날짜함수에 문자열이 오면(`LAST_DAY(P || '01')`) `asDate()`로 자동 변환.
- **따옴표 정규화**: 입력값 양끝 따옴표 1쌍 제거(`unq`/`stripq`) → 사용자가 `'202606'`/`202606` 아무렇게나 넣어도 동작(치환은 자동 인용, 동적은 원본이 인용 제공).
- **node 테스트**: `test.js`/`test-dynamic.js`로 모든 실제 프로시저 패턴을 회귀 검증(브라우저 없이).

### 해결한 버그
- **복사 콜백 `currentTarget` null**: `.then/.catch`(비동기)에서 `e.currentTarget`을 읽어 null 참조 → 버튼 참조를 동기로 `const btn=e.currentTarget`로 잡아 해결. (+클립보드 거부 시 execCommand 폴백)
- **배포해도 안 바뀜**: 브라우저가 `app.js`를 캐시 → `index.html`에서 `app.js?v=Date.now()`로 캐시버스팅.
- **누락 괄호 등 파싱 실패**: 계산 실패한 파생칸에 `⚠ 이유`를 placeholder로 표시(조용한 빈칸 → 원인 노출).
- **버튼 텍스트 변경 시 UI 밀림(가로+세로)**: '복사됨 ✓'의 ✓ 이모지가 폭/줄높이 키움 → `#copy/#reset` 폭·높이 고정 + inline-flex 중앙정렬.

### 실패/폐기한 방법
- **라바 goo `contrast(15~20)`**: 화면이 까맣게 사라지거나(투명 배경+과대비) 눈알처럼 징그러움 → 실패.
- **라바 흡수(중력처럼 끌어당김)**: 방울이 서로 "달려들어" 게임처럼 확 빨아먹음 → 부자연스러움, 폐기.
- **라바 클릭 분리(임펄스/쌍둥이/독립 파티클 시뮬)**: 여러 번 시도했으나 "완전히 안 갈라짐/징그러움" → 전부 제거하고 **단색 goo + 자유 부유**만 남김.
- **GitHub API로 버전 자동표기**: 사용자 망분리 환경에서 API 불가 → 수동 문자열 `const VER='v1.5.3'`로 회귀.
- **복사 애니메이션 지니/진공청소기 warp**: CSS로 진짜 genie warp 불가, 어색함 → 첫 버전(축소·회전 후 버튼으로 흡입)으로 복귀.

## 주요 결정 사항

- **파일 분리(app.js/dynamic.js)**: 로직을 DOM 없는 모듈로 빼서 브라우저(`<script>`)와 node 테스트가 동일 코드 사용. 동적 모드는 걷어내기 쉽게 `dynamic.js`로 격리(원래 별도 페이지였다가 "페이지 이동 싫다"는 요청으로 index에 자동감지 통합).
- **정적 사이트 유지(백엔드 X)**: GitHub Pages/`file://`에서 동작해야 해서 순수 HTML+JS, 외부 라이브러리 0.
- **날짜는 UTC 고정**: DST 드리프트 방지(`Date.UTC`).
- **변수 추출은 `P_`/`V_` 접두사 기준**: 범용 스캔은 컬럼/키워드 오탐 → 접두사 + 사용자 등록 방식으로. (동적 모드는 대입/조건에서 참조된 식별자로 판별)
- **동적 FOR/LOOP 미지원 결정**: CONNECT BY 커서 루프는 DB 실행이 필요 → 범용 불가, 명시적으로 건너뜀.
- **테마는 라바 단색 goo**: 다색/가짜입체는 징그러움 → 한 색 + goo + 부유만.

## 주의사항 & 교훈

- **커밋 메시지 규칙**: 사용자 전역 설정(attribution 비활성)에 따라 `git -c commit.gpgsign=false commit` 사용, Co-Authored-By 등 붙이지 말 것.
- **금지 표현**: "박다/박아넣다" 류 쓰지 말 것(사용자 전역 지침). "넣다/추가/반영" 사용.
- **배포 = 커밋+push**. Pages는 CDN 전파에 1~2분. "안 바뀐다" 신고는 대개 캐시 → 강력새로고침 안내. 라이브 확인은 `curl --ssl-no-revoke`(Windows schannel 폐기검사 회피).
- **캐시버스팅 유지**: `app.js?v=`, `dynamic.js?v=` 빼지 말 것(안 그러면 배포 반영 안 됨 신고 재발).
- **시각/UX는 수치 튜닝 지옥**: "몽환적/라바 느낌" 같은 추상 요청은 한 번에 한 파라미터씩 고치면 사용자가 매우 답답해함. **방향을 먼저 확정(AskUserQuestion)**하고 한 번에 제대로 갈 것. 이번에 라바로 수십 왕복함 — 반복 금지.
- **버전 올리기**: 배포마다 `index.html`의 `VER`와 `<div id="ver">` 둘 다 갱신(sed로 일괄). 의미형(메이저.마이너.패치): 버그=패치, 기능=마이너, 개편=메이저.
- **GateGuard 훅**: 파일 생성/첫 Bash 전에 fact 4종 제시 요구가 뜸 — 당황 말고 사실 적고 재시도.
- **Windows 줄끝 경고(LF→CRLF)**: 정상, 무시.

## 중요 파일 맵

| 파일 | 역할 |
|------|------|
| `index.html` | 화면(UI) + 두 모드 자동 감지 + UI 로직(analyze/convert/recompute/추천버튼/＋팝업/복사·초기화/버전/배경토글·오로라·라바 애니메이션). CSS 인라인. |
| `app.js` | 치환 엔진(브라우저/Node 공용): 토크나이저·Oracle 식 평가기(`evalExpr`)·`literalOf`·`parseInit`·`scanVars`·`freeVars`·`substitute`·`unq`. |
| `dynamic.js` | 동적 SQL 조립: `parseProgram`(IF/ELSIF/ELSE/END IF 파서)·`assemble`(분기선택+조립, append 줄바꿈)·`evalCond`(조건평가)·`condDefaults`(IF값 기본프리필)·`mainVar`. `evalExpr`는 app.js 재사용. |
| `test.js` | 치환 엔진 회귀 테스트(`node test.js`) — 실제 프로시저 6종 패턴. |
| `test-dynamic.js` | 동적 조립 테스트(`node test-dynamic.js`). |
| `README.md` | 사용법/지원범위/동적모드/옵션 설명. |
| `.gitignore` | `*.zip`, `node_modules/`. |

### 로컬 검증
```
node test.js
node test-dynamic.js
```
`index.html`은 더블클릭(`file://`)으로도 동작. 배경 토글/복사는 https(Pages)에서 완전 동작.
