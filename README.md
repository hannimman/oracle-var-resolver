# oracle-var-resolver

Oracle 프로시저의 **변수 초기화 부분을 브라우저에서 직접 계산**해서, 쿼리 안의 변수를 실제 값(SQL 리터럴)으로 치환해주는 정적 웹 페이지.

프로시저 일부 쿼리만 떼어다 다른 쿼리창에서 바로 돌려보고 싶을 때, 상단 변수 세팅(주로 날짜 계산)을 손으로 풀던 노가다를 없애는 도구다.

## 데모 (GitHub Pages)

https://hannimman.github.io/oracle-var-resolver/

## 쓰는 법

1. **① 쿼리** 칸에 변수(`V_...`, `P_...`)가 들어간 쿼리를 붙여넣는다.
2. **② 변수 초기화 블록** 칸에 `VAR := EXPR;` 또는 `SELECT ... INTO VAR FROM DUAL;` 형태의 초기화 부분을 붙여넣는다.
   - 초기화 블록을 **비워두면**, 쿼리에서 `P_`/`V_` 로 시작하는 변수를 자동으로 찾아 입력칸을 만든다.
3. **분석 ▶** 을 누른다.
4. **입력 변수**(예: `P_PROC_YM`)에 값을 넣으면, **파생 변수**(`V_PROC_DATE` 등)가 자동으로 계산된다.
   - 모든 칸은 직접 수정 가능. 계산이 안 되는 값(테이블 조회 등)은 직접 입력한다.
5. **변환 ▶** 을 누르면 쿼리의 변수가 값으로 치환된 결과가 나온다. **결과 복사** 로 가져간다.

### 입력 값 규칙

- 입력 변수는 `202606` 또는 `'202606'`(따옴표째) 둘 다 받는다. 결과에는 `'202606'` 로 들어간다.
- 날짜 리터럴은 시각이 있으면(`23:59:59` 등) 시간까지, 없으면 날짜만 출력한다.
  - `TO_DATE('2026-06-01','YYYY-MM-DD')`, `TO_DATE('2026-06-30 23:59:59','YYYY-MM-DD HH24:MI:SS')`

## 지원 범위

- **함수**: `TO_DATE`(포맷 생략 가능), `TO_CHAR`, `LAST_DAY`, `ADD_MONTHS`, `SUBSTR`, `REPLACE`
- **연산**: 날짜 ± 정수(일), 문자열 `||` 연결, 괄호식
- **마스크**: `YYYY/MM/DD/HH24/MI/SS` (대소문자 무시), 구분자 없는 문자열(`'20260601'`)도 매칭
- **대입 형태**: `VAR := EXPR;`, `SELECT EXPR INTO VAR FROM DUAL;`
- **테이블 조회**(`SELECT COUNT(*) INTO VAR FROM 테이블`)는 계산 불가 → "수동입력"칸으로 표시

> 변수 추출은 `P_`/`V_`(대소문자) 접두사 기준. 다른 이름은 초기화 블록에 정의하거나 그 접두사를 쓰면 된다.
> 지원 함수 밖의 식이 나오면 그 변수칸만 비고 직접 입력하면 된다.
